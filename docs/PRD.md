# 食譜收藏夾 PRD 產品需求文件

## 1. 專案概述

### 背景與動機
現代人常在網路上尋找食譜，但散落各處的食譜難以統一管理。使用者在料理過程中需要在不同網站或 APP 之間跳轉，且無法加入個人實作筆記與食材清單。為了滿足下廚愛好者整理與管理個人菜單的需求，建立專屬的「食譜收藏夾」，打造個人化的雲端私房菜譜。

### 目標用戶
- **家庭煮夫/煮婦**：需要管理大量日常菜單並快速檢索的人。
- **料理新手**：習慣在網路尋找靈感，並希望記錄實作筆記的人。
- **飲食管理群體**：需要依據食材成分或營養標籤篩選食譜的人。

### 核心價值主張
這是一個專為下廚愛好者設計的數位食譜管理工具，協助使用者從各大網站快速抓取、整理並標籤化儲存心儀食譜，提升備料與料理時的效率與體驗。

## 2. 功能需求

### 1. 一鍵網頁抓取
- **使用者故事**：作為一名料理新手，我希望能夠透過輸入食譜網址自動抓取網頁內容並儲存，以便快速將不同網站的食譜集中到我的收藏夾。

### 2. 智慧標籤與分類系統
- **使用者故事**：作為一名飲食管理群體，我希望能夠為食譜加上食材、熱量、難易度等自訂標籤，以便日後依據特定需求快速篩選出合適的食譜。

### 3. 互動式烹飪模式
- **使用者故事**：作為一名家庭煮夫/煮婦，我希望能在烹飪時進入適合螢幕瀏覽的大字體互動式烹飪模式，以便在雙手沾滿食材時仍能清晰地照著步驟操作。

### 4. 食材清單自動生成
- **使用者故事**：作為一名家庭煮夫/煮婦，我希望系統能將選定食譜的食材自動整合成一份待買清單，以便我去超市採購時能一次買齊、不遺漏。

### 5. 個人化實作筆記與評分
- **使用者故事**：作為一名料理新手，我希望能夠在每次實作後對食譜留下個人筆記與評分，以便記錄調整口味的心得，作為下次烹飪的參考。

## 3. 非功能需求

### 技術限制
- 前端：HTML、CSS (Vanilla)、JavaScript
- 後端：Flask (Python)
- 模板引擎：Jinja2
- 資料庫：SQLite

### 效能與安全考量
- **效能**：一鍵抓取網頁食譜的操作需在合理時間內完成（例如 3 秒內），避免使用者等待時間過長。
- **安全**：需防範 XSS 攻擊，確保使用者輸入的筆記或抓取到的內容在經過適當編碼後呈現；防範 SQL Injection；需實作基本的使用者身份驗證 (Authentication) 保護個人私房食譜隱私。

## 4. MVP 範圍

### Must Have (必須具備)
- 使用者註冊與登入
- 建立、檢視、編輯、刪除食譜（基本 CRUD 功能）
- 一鍵網頁抓取基本功能（支援通用食譜網站的標題、內文與圖片）
- 食譜標籤與簡單分類系統
- 個人實作筆記

### Should Have (應該具備)
- 食材清單自動生成（可自行勾選購買狀態）
- 食譜評分系統
- 互動式烹飪模式（大字體、步驟逐條顯示）

### Nice to Have (加分項目)
- 營養成分分析總覽
- 依據剩餘食材推薦食譜
- 支援防休眠或語音控制切換步驟
